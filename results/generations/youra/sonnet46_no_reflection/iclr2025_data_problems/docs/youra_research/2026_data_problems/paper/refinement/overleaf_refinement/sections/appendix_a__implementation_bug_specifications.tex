
\subsubsection{A.1 DCPDDDetector Log-Ratio Fix (Bug B3)}

The executed implementation computes \texttt{score = -ref_log_prob} using a single model. The correct implementation requires two models and computes the per-token log-likelihood ratio:

```python
score = log_prob_target - log_prob_reference  \# per token, then aggregate

```

Required models: Pythia-6.9B (target), Pythia-2.8B (reference).

\subsubsection{A.4 StratifiedEvaluator F1 Matrix Fix (Bug B2)}

The \texttt{f1_matrix} computation uses scalar broadcast, causing \texttt{indeterminacy_rate} to be degenerate (0.000). The correct implementation requires per-item F1 computation followed by per-item margin computation between the top-1 and top-2 detectors.
