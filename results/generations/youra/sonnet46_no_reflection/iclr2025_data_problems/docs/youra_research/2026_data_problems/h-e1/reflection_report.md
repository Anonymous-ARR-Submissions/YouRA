## What Failed

### 1. Geometry Stratum Collapse
All benchmark items (25,403 total) were assigned exclusively to the `lexical` stratum. No `semantic` or `indeterminate` items were produced. Root cause: SBERT cosine similarities for benchmark items against streaming corpus documents are uniformly near-zero (non-contaminated items have low embedding similarity to random corpus samples). The 75th percentile threshold mechanism collapsed when the embedding score distribution was degenerate, placing all items below the semantic stratum threshold.

### 2. N-gram Lexical Recall Below 0.80
GSM8K (math problems, 1,319 items) showed 0.0 n-gram recall against all 3 corpora. This is expected — math step-by-step reasoning has no verbatim overlap with general text corpora. This pulls the overall 9-pair average below the 0.80 threshold (5/9 pairs hit recall=1.0).

### 3. Min-K%++ F1 Variance = 0.000
All minkpp detector predictions were 0 across all benchmark-corpus pairs. Root cause: the DCPDDDetector (task-011) uses `-ref_log_prob` instead of the true log-likelihood ratio, and ConStatDetector (task-012) uses a custom heuristic rather than the `llmsanitize.contamination.constat()` API. These implementation gaps, flagged by the validator, caused LLM-based detectors to produce trivially zero predictions.

## Root Cause Analysis

1. **Stratum collapse is a fundamental algorithmic issue**: The 75th percentile stratification cannot distinguish contaminated from non-contaminated items when the corpus is too large/diverse for benchmark items to have meaningful embedding similarity to random corpus samples. The hypothesis requires ground-truth-contaminated items to test.

2. **N-gram recall criterion misaligned with benchmark type**: A math benchmark (GSM8K) will never have verbatim overlap with general pretraining text. The success criterion needs to be benchmark-specific or exclude math-style benchmarks.

3. **LLM detector implementations incomplete**: DCPDDDetector and ConStatDetector have correctness issues that prevent meaningful F1 variance measurement. Even if fixed, the lack of semantic/indeterminate strata would make variance = 0.

## Lessons Learned

1. Stratification requires ground-truth-labeled contaminated/non-contaminated items, not just corpus proximity scores
2. Benchmark heterogeneity (NLU vs. math) requires separate treatment in recall criteria
3. The PoC should use injected synthetic contamination to validate the geometry decomposition before testing on real corpora
4. SBERT similarity on streaming random corpus samples is not a reliable proxy for contamination geometry — need targeted retrieval (top-k similar docs per benchmark item)

## Routing Decision

**ROUTED_TO_PHASE_0**: The hypothesis h-e1 requires fundamental reassessment. The existence claim cannot be validated with the current experimental design. Key redesign needs:
- Use injected contamination (controlled synthetic data) to verify stratum formation
- Use top-k corpus retrieval (not random streaming) for geometry feature computation
- Separate benchmark-type-specific recall criteria
