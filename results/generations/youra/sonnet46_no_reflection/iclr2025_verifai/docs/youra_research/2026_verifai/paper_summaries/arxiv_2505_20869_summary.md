# Step-Wise Formal Verification for LLM-Based Mathematical Problem Solving

## Key Metadata
- **Authors:** Kuo Zhou, Lu Zhang et al.
- **Year:** 2025
- **Venue:** arXiv (preprint)
- **Core Contribution:** Proposes MATH-VF, a training-free framework combining a formal language translation step with tool-integrated verification to perform step-wise formal verification and refinement of LLM-generated mathematical problem solutions.

## Section Summaries

### Abstract
Large Language Models (LLMs) have demonstrated formidable capabilities in solving mathematical problems, yet they may still commit logical reasoning and computational errors during the problem-solving process. Thus, this paper proposes a framework, MATH-VF, which includes a Formalizer and a Critic, for formally verifying the correctness of the solutions generated by large language models. Our framework first utilizes a Formalizer which employs an LLM to translate a natural language solution into a formal context. Afterward, our Critic (which integrates various external tools such as a Computer Algebra System and an SMT solver) evaluates the correctness of each statement within the formal context, and when a statement is incorrect, our Critic provides corrective feedback. We empirically investigate the effectiveness of MATH-VF in two scenarios: 1) Verification: MATH-VF is utilized to determine the correctness of a solution to a given problem. 2) Refinement: When MATH-VF identifies errors in the solution generated by an LLM-based solution generator for a given problem, it submits the corrective suggestions proposed by the Critic to the solution generator to regenerate the solution. We evaluate our framework on widely used mathematical benchmarks: MATH500 and ProcessBench, demonstrating the superiority of our approach over existing approaches.

### Introduction & Motivation
Applying LLMs to mathematical reasoning is critical but error-prone, especially with complex reasoning and calculations. Existing verification methods either rely on informal natural language verification that struggles with fine-grained errors, or formal verification that is challenging due to the complexity of fully formalizing informal solutions and poor usability of formal proofs as feedback. Autoformalization approaches often only formalize problems, not intermediate solving steps, limiting error detection. This paper addresses these gaps by proposing MATH-VF, which jointly formalizes the entire solution using a new first-order-like formal language, SimpleMath, and verifies each step with integrated external solvers. This approach offers stepwise verification, useful feedback, and computational efficiency by leveraging the sparsity in solution steps.

### Methodology
MATH-VF is a two-component framework: the **Formalizer** and the **Critic**.

- **Formalizer:**  
  Uses few-shot prompting of LLMs (e.g., Deepseek-v3, GPT-4) with detailed grammar instructions to translate natural language problem solutions into a formal language, *SimpleMath*. SimpleMath extends classical first-order logic with added syntactic sugar for convenient mathematical expression. For example, definitions like the Fibonacci function are encoded with case distinctions in SimpleMath resembling natural language forms.  
  The Formalizer produces a Fitch-style formal context comprising statements classified as Facts, Assumptions, Theorems, Definitions, and Conclusions.  
  Each statement is a node in a *Solution Graph* \( G = (V, E) \), where nodes correspond to statements, and edges represent "direct conclusions." This graph is typically sparse, meaning each conclusion depends on only a few premises. This sparsity allows significant reduction in input size when verifying individual steps.

- **Critic:**  
  The Critic verifies the correctness of judgments \( T_i \vdash Q_i \), where \( T_i \) is a set of premises and \( Q_i \) is a conclusion, derived from the formal context. Instead of verifying the entire proof globally, the Critic checks each step independently using both LLM reasoning and calls to external tools:
  - **SymPy:** For symbolic computations such as differentiation, integration, solving equations.
  - **Z3 SMT Solver:** For logic satisfiability checking involving arithmetic, arrays, datatypes, and nonlinear constraints.  
  The Critic adopts a prompt-based few-shot pipeline that separately reasons and invokes these tools via specialized LLM-based agents rather than generating executable code outright.

- **Efficiency via Sparsity:**  
  Traditional methods input a growing number of premises per step, with complexity \( C_1(n) = \frac{n(n+1)}{2} \) for \( n \) steps. MATH-VF exploits sparsity such that the maximum premises per conclusion \( M \leq 4 \), yielding \( C_2(n) \leq n \times M \), drastically reducing token count and computational effort.

- **Solution Refinement:**  
  If verification fails, MATH-VF feeds the Critic's feedback and the solution back to the solution Generator (e.g., Deepseek-v3) to regenerate an improved solution. This iterative loop continues until a correct solution is found or a maximum iteration limit is reached. This leverages constructive feedback from external tools to surpass self-refinement methods that rely only on LLM internal evaluation.

### Experiments & Results
- **Datasets:**  
  - *MATH500*: A benchmark for math problem-solving requiring formal verification.  
  - *ProcessBench*: Contains problems from GSM8K, MATH, OlympiadBench, and Omni-MATH, covering diverse difficulty levels.

- **Baselines:**  
  - LLM + Coq: Formalizes solutions into Coq and verifies with the interactive theorem prover.  
  - Primary Critic: An LLM-only critic model without external tool integration.  
  - Qwen2.5-MATH-PRM-72B: A process reward model (PRM)-based baseline requiring training.  
  - Self-consistency and self-refinement methods for solution selection and iterative improvement.

- **Metrics:**  
  - Accuracy in determining solution correctness.  
  - F1-score on ProcessBench for stepwise correctness evaluation.  
  - Solution identification accuracy among candidates.  
  - Improvement in solution accuracy via refinement.

### Discussion & Conclusion
MATH-VF introduces a novel stepwise formal verification framework that bridges informal LLM outputs and formal mathematical verification using a human-like formal language and external symbolic tools. It demonstrates improved verification, identification, and refinement performance over various baseline approaches, notably without requiring additional training. Limitations include dependence on correct syntax generation by the Formalizer and the inability to assess the effectiveness of reasoning steps beyond correctness. Future work could incorporate syntax correction parsers and evaluate step effectiveness for further enhancement.

## Key Contributions

- Presented MATH-VF, a training-free, iterative formal verification and solution refinement framework demonstrating superior robustness and efficiency across multiple math benchmarks.
