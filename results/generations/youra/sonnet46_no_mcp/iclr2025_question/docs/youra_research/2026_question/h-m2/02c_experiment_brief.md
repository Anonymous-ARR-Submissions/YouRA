## Implementation Research Summary

### Archon Knowledge Base Findings

**MCP Status:** Unavailable (TEST no-MCP environment). LLM-native reasoning applied.

**Query 1: NLI clustering for semantic uncertainty — experiment design**
- The `lorenzkuhn/semantic_uncertainty` repository (Kuhn et al. 2023) implements NLI-based semantic clustering for LLM responses using deberta-large-mnli. The official implementation uses bidirectional NLI: two responses are in the same cluster if NLI(r_i → r_j) = ENTAILMENT AND NLI(r_j → r_i) = ENTAILMENT (bidirectional entailment).
- Typical experiment: N stochastic samples → NLI pairwise comparison → connected components → cluster count → cluster entropy as UQ signal
- Known behavior: On factual QA tasks (TriviaQA, NQ), mean cluster counts are typically 2–4 for N=5 samples, with aggregation rates of 60–80%

**Query 2: NLI clustering implementation challenges**
- Key challenge: deberta-large-mnli was fine-tuned on MNLI (formal NLI) not on QA response pairs. Short QA responses ("Paris", "The capital of France is Paris") may not trigger entailment reliably.
- Best practice: Use the official lorenzkuhn implementation which handles edge cases (single-word responses, empty responses)
- Common pitfall: Incorrect entailment threshold — official implementation uses argmax of [contradiction, neutral, entailment] logits
- Degenerate case: If model generates nearly identical samples (low temperature diversity), all N responses are semantically equivalent → cluster_count=1 for all → full collapse

**Query 3: Cluster count distribution benchmarks**
- Kuhn et al. (2023) report that semantic entropy effectively clusters TriviaQA responses: typical aggregation rate ~70% for N=5 at temperature=1.0
- H-E1 H-M1 findings: mean_clusters=4.644 (close to 5) suggests most examples have 4–5 distinct clusters → low aggregation — this is the degenerate case diagnosed in H-M1

```python
# From lorenzkuhn/semantic_uncertainty (Kuhn et al. 2023)
# NLI-based semantic equivalence clustering

from transformers import AutoModelForSequenceClassification, AutoTokenizer
import torch

def get_semantic_ids(strings_list, model, tokenizer, strict_entailment=False, device='cuda'):
    """Assign semantic cluster IDs using NLI bidirectional entailment."""
    semantic_set_ids = {}
    with torch.no_grad():
        for i, string1 in enumerate(strings_list):
            if i not in semantic_set_ids:
                semantic_set_ids[i] = i  # initialize own cluster
            for j in range(i + 1, len(strings_list)):
                string2 = strings_list[j]
                # Bidirectional NLI check
                inputs_forward = tokenizer(string1, string2, return_tensors='pt').to(device)
                inputs_backward = tokenizer(string2, string1, return_tensors='pt').to(device)
                logits_f = model(**inputs_forward).logits
                logits_b = model(**inputs_backward).logits
                # entailment class is index 2 for deberta-large-mnli
                entail_f = torch.argmax(logits_f, dim=-1).item() == 2
                entail_b = torch.argmax(logits_b, dim=-1).item() == 2
                if entail_f and entail_b:
                    semantic_set_ids[j] = semantic_set_ids[i]
    return list(semantic_set_ids.values())
```

**Pattern 2: Cluster count computation and aggregation rate**
```python
def compute_cluster_stats(cluster_ids_per_example):
    """Compute aggregation rate and collapse rate from cluster ID lists."""
    cluster_counts = [len(set(ids)) for ids in cluster_ids_per_example]
    n = len(cluster_counts)
    aggregation_rate = sum(c < 5 for c in cluster_counts) / n  # < N=5
    collapse_rate = sum(c == 1 for c in cluster_counts) / n    # all same
    mean_count = sum(cluster_counts) / n
    return {
        'mean_cluster_count': mean_count,
        'aggregation_rate': aggregation_rate,
        'collapse_rate': collapse_rate,
        'cluster_counts': cluster_counts
    }
```

```python
# Core Mechanism: NLI Semantic Clustering Analysis
# Based on: lorenzkuhn/semantic_uncertainty (Kuhn et al. 2023)
# H-M2 Analysis: Cluster count distribution on HaluEval-QA (2,000 examples, N=5)

import json
import numpy as np
from scipy.stats import pointbiserialr

def analyze_cluster_distribution(h_e1_outputs_path, halueval_labels_path):
    """
    Analyze NLI cluster count distribution from H-E1 outputs.

    Input:  H-E1 NLI clustering results (cluster_counts per example)
            HaluEval binary labels (hallucinated=1, factual=0)
    Output: aggregation_rate, collapse_rate, correlation stats
    """
    # Step 1: Load H-E1 cluster counts (already computed)
    with open(h_e1_outputs_path) as f:
        h_e1_data = json.load(f)
    cluster_counts = np.array([ex['cluster_count'] for ex in h_e1_data])  # shape: (2000,)

    # Step 2: Load HaluEval binary labels
    with open(halueval_labels_path) as f:
        labels = np.array(json.load(f))  # 1=hallucinated, 0=factual

    N = 5  # stochastic samples per example
    n = len(cluster_counts)

    # Step 3: Compute aggregation rate (primary success criterion)
    aggregation_rate = np.mean(cluster_counts < N)   # fraction with < 5 clusters
    collapse_rate = np.mean(cluster_counts == 1)      # fraction with all-same cluster
    mean_count = np.mean(cluster_counts)
    std_count = np.std(cluster_counts)

    # Step 4: Correlation between cluster_count and hallucination label
    # Lower cluster_count → more semantic agreement → expected for factual (label=0)
    r_pb, p_value = pointbiserialr(labels, cluster_counts)

    # Step 5: Stratify by question type if available
    return {
        'mean_cluster_count': float(mean_count),
        'std_cluster_count': float(std_count),
        'aggregation_rate': float(aggregation_rate),
        'collapse_rate': float(collapse_rate),
        'correlation_cluster_label': float(r_pb),
        'correlation_p_value': float(p_value),
        'n_examples': n
    }
```
