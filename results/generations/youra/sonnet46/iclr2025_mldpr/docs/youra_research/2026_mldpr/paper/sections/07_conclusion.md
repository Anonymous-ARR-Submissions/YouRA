# 7. Conclusion

We began by observing a striking fact: across 758 ML datasets audited from HuggingFace Hub and OpenML, not a single dataset documented its preprocessing steps in machine-readable form, and none documented their intended uses or known limitations. These are the sections that responsible AI documentation frameworks designate as highest priority. The puzzle motivating this paper was whether this absence reflects genuine documentation failure or something more structural — and whether an automated pipeline could reveal the answer at population scale.

Our answer is both. The automated DTS-weighted scoring pipeline we developed achieves 91.8% corpus coverage (n=758) with high internal consistency (proxy r=0.989, p<10⁻¹⁰⁰), confirming that automated cross-repository documentation scoring is technically feasible from public ML dataset APIs. But the pipeline's own per-section results reveal the structural explanation: Preprocessing and Uses documentation scores near-zero universally not because creators avoid writing it, but because these sections exist as free-text prose in dataset cards — and structured repository APIs do not index prose as machine-readable fields. The documentation API gap is not a gap in creator effort; it is a gap in infrastructure design.

**Summary of Contributions.** This work contributes:

1. *The first automated cross-repository DTS-weighted documentation scoring pipeline*, achieving 91.8% coverage across 758 datasets from HuggingFace Hub and OpenML — scaling manual prior work by 7.58× and extending it cross-repository. The pipeline is open-source and immediately deployable.

2. *Large-scale empirical confirmation of the DTS section asymmetry*: Preprocessing (0.002) and Uses (0.000) coverage is near-zero across all repositories in automated API-based scoring, consistent with Rondina et al.'s [2025] manual findings at 7.58× scale. DTS inverse-frequency weighting compresses the overall score by 22% relative to naive counting, demonstrating that simple field presence rates systematically overestimate effective documentation quality.

3. *Identification of the documentation API gap*: the inverse relationship between a documentation section's importance in responsible AI frameworks and its accessibility through structured repository API endpoints. The sections that matter most for responsible AI deployment are the ones that automated scorers cannot measure from current API infrastructure.

**Future Directions.** The pipeline infrastructure established here enables a clear causal research agenda. First, fixing the UCI field mapping gap and running ANOVA + DiD analysis (H-M1) will test whether HuggingFace's 2021 YAML adoption caused higher documentation completeness — transforming our descriptive cross-sectional findings into causal evidence. Second, testing whether DTS completeness predicts dataset download adoption via pre-registered negative binomial regression (H-M3) will determine whether documentation quality is not just a compliance metric but a driver of dataset reuse. Third, conducting human annotation validation (n=120, stratified) will establish external validity for automated DTS scores — resolving whether the proxy consistency demonstrated here (r=0.989) translates to agreement with expert human judgment.

The documentation API gap itself points to two infrastructure interventions worth pursuing: repository operators could add structured API fields for Preprocessing and Uses sections, dramatically improving automated compliance assessment; and LLM-based parsing of full dataset card prose could recover completeness signals from sections that structured APIs currently miss. Either path would bring automated documentation quality assessment closer to what responsible AI frameworks actually require.

We began with a finding about absence — zero Preprocessing, zero Uses, at population scale. We end with a finding about infrastructure: the absence is measurable, its cause is identifiable, and the path to closing it is now clear. Automated documentation quality assessment at scale is not a solved problem, but it is a tractable one, and the measurement infrastructure to pursue it is now in place.
