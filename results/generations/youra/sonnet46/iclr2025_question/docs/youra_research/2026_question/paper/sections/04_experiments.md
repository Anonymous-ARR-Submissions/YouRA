# Experimental Setup

## Research Questions

Our experiments are designed to directly test the claims arising from the commission/omission detection insight. Four research questions structure the evaluation:

- **RQ1 (Existence)**: Does frozen NLI contradiction scoring achieve above-chance AUROC on HaluEval commission tasks (dialogue, QA)?
- **RQ2 (Mechanism)**: Is the detection signal a structural property of DeBERTa's graded support sensitivity, or an artifact of the score distribution?
- **RQ3 (Comparison)**: Does generation-free NLI match or exceed generation-based SelfCheckGPT-NLI on the same benchmark?
- **RQ4 (Boundary)**: Does the approach fail on omission-type tasks (summarization), and can this failure be mechanistically explained?

## Dataset

**HaluEval** [Li et al., 2023] is the primary evaluation benchmark. We use the official test splits for all three tasks:

| Task | Context Type | Response Type | Pairs | Label Balance |
|------|-------------|---------------|-------|---------------|
| Dialogue | Multi-turn conversation (last 3 turns) | Single response turn | 20,000 | 50/50 |
| QA | Question + knowledge passage | Answer sentence | 20,000 | 50/50 |
| Summarization | Source document | Abstractive summary | 20,000 | 50/50 |

HaluEval hallucination examples were generated by prompting GPT-3.5 to produce plausible-but-incorrect responses to the same context as the ground-truth responses. Non-hallucinated examples are the original model-generated responses. This construction yields controlled, paired examples suitable for binary hallucination detection evaluation.

We use the HuggingFace version (`pminervini/HaluEval`) which provides all three task splits in a unified format. Labels are binary: 1 = hallucinated, 0 = non-hallucinated.

**Rationale for HaluEval**: HaluEval is the de facto standard multi-task hallucination evaluation benchmark, covering the three most practically important LLM application domains. Its 20,000-pair scale per task provides statistical power sufficient for DeLong's test to detect AUROC differences of 0.005 or greater (estimated from standard error calculations). No other publicly available benchmark provides comparable scale and task diversity with binary hallucination labels.

## Baseline: SelfCheckGPT-NLI

We compare against SelfCheckGPT-NLI [Manakul et al., 2023] as the primary generation-based baseline. SelfCheckGPT-NLI generates multiple stochastic samples from the same prompt and uses an NLI model to measure consistency between the generated response and each sample — hallucinated content is expected to be inconsistently reproduced. This baseline directly tests our central claim: that generation-free NLI achieves competitive detection performance without the sampling overhead.

**Baseline AUROC values** (from prior experiment results on the same HaluEval test split):
- Dialogue: 0.48
- QA: 0.53
- Summarization: not reported (method inapplicable under our experimental constraints)

**Rationale for this baseline**: SelfCheckGPT is the most widely cited and replicated sampling-based hallucination detection method, and its NLI variant uses the same underlying signal (NLI consistency) as our approach, making the comparison methodologically clean. The difference in AUROC measures the value of generation-free post-hoc application vs. sampling-based consistency checking.

## Evaluation Metrics

**Primary: AUROC** — Area Under the ROC Curve. Threshold-independent discrimination measure suitable for binary classification with balanced labels. AUROC = 0.5 corresponds to random ranking; AUROC = 1.0 is perfect.

**Statistical significance: DeLong test** [DeLong et al., 1988] — Non-parametric variance estimation for AUROC statistics. We test each task's AUROC against the 0.5 null hypothesis at α = 0.05. A task *passes* the gate if AUROC > 0.55 with DeLong p < 0.05.

**Gate criterion**: The approach is declared to *exist* (h-e1 PASS) if ≥ 2/3 tasks satisfy the gate criterion. This conservative multi-task gate ensures the result is not driven by a single favorable task.

**Effect size: Cohen's d** — Standardized mean difference in contradiction scores between hallucinated and non-hallucinated responses. Cohen's d > 0.5 indicates medium-to-large practical effect.

**Mechanism tests (secondary)**:
- *Wilcoxon rank-sum test*: Stochastic dominance of hallucinated scores over non-hallucinated scores. Significance threshold p < 0.05.
- *KL divergence from uniform*: Distributional concentration of per-class scores. Threshold KL > 0.05.

## Implementation Details

- **Model**: `cross-encoder/nli-deberta-v3-large` (304M parameters, frozen, float32)
- **Hardware**: Single NVIDIA H100 NVL, CUDA_VISIBLE_DEVICES=0
- **Batch size**: 64 pairs per forward pass
- **Max sequence length**: 512 tokens (truncation applied; verified minimal impact given context windowing)
- **Label map verification**: `verify_label_map()` confirms contradiction class index = 0 for this model
- **No training**: All parameters frozen; no gradient updates; inference-only
- **Net-contradiction score**: `P(contradiction) - P(entailment)` per sentence, then max aggregation for dialogue/QA
- **Total inference pairs**: 60,000 (20,000 × 3 tasks)
- **Total inference time**: approximately 3.5 hours (H100 NVL)
