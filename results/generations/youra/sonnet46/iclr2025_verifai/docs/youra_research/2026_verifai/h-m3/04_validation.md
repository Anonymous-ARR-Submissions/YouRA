### Lessons Learned

**What Worked:**
- `output_logits=True` with `max_new_tokens=1` is a clean, reliable logprob extraction approach
- Leading space in " True"/" False" token encoding is critical for correct tokenization
- Checkpoint/resume at 100-pair intervals made the experiment resilient
- Wide-format CSV parsing (h-m2 actual output) — confirmed the data_loader needs to handle this
- H100 GPU: ~4 minutes for 5,730 pairs total (3 models) — highly efficient

**What Didn't Work / Unexpected Issues:**
- `vocab_size_correct` indicator has a shape-checking bug (returns False even when vocab is correct); does not affect gate evaluation
- h-m2 `tier_assignments.csv` is wide format (per-model tier columns), not long format — needed data_loader fix
- Solutions in `h-e1/results/` (not `h-m1/results/` as documented in some specs) — `DEFAULT_HM1_RESULTS` must point to h-e1

**Unexpected Findings:**
- deepseek_6.7b shows highest mean_c (0.648) and strongest std(c) (0.0781) — most confident model
- codellama_7b shows lowest mean_c (0.368) — most conservative model
- llama3_8b shows weakest but still positive correlation with correctness (r=0.197) — useful signal for h-m4 ECE
- deepseek_6.7b shows near-zero correlation (-0.046, p=0.048) — confidence less aligned with correctness

**Key Insight:** P(True) logprob extraction is validated as a working mechanism. All 3 models produce non-degenerate confidence distributions. The confidence signal is statistically significant (p<0.05 for all models) but weak (r=0.14–0.20), which is consistent with the main hypothesis that difficulty-stratified ECE will reveal systematic miscalibration.

### Recommendations for h-m4 (MECHANISM: ECE per difficulty tier)

**General:**
1. Load `ptrue_confidence_scores.json` directly from h-m3/results — no need to re-run inference
2. Use `correctness_*.json` from h-e1/results for binary correctness labels
3. Use `tier_assignments.csv` from h-m2/results for hard/easy tier assignments

**Specific:**
- ECE M=15 bins: use `np.linspace(0, 1, 16)` for bin edges
- Bootstrap CI: 1000 samples on DELTA_ECE = ECE(hard) - ECE(easy)
- Temperature scaling: fit T on 20% holdout from hard+easy combined
- Expected DELTA_ECE: h-m3 shows hard mean_c (0.366–0.652) vs easy mean_c (0.385–0.644) — small but potentially systematic

**Warnings:**
- deepseek_6.7b confidence is notably higher (mean=0.648) — may need model-specific calibration
- codellama_7b has very few easy pairs (n=185 easy vs n=1705 hard) — unbalanced tier split; note in ECE analysis
