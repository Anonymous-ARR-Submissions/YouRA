Observed conventions: Python packages live under `src/`; configuration is centralized in dataclass-style config modules; functions/classes use snake_case / PascalCase naming; files include concise docstrings and explanatory comments around nontrivial logic; outputs are typically written as JSON/CSV plus markdown validation reports and figures. Shell scripts handle environment activation, dependency installation, GPU selection, and then call `python -m src.main` or related entrypoints.
