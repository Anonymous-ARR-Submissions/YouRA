# Conclusion

We opened by noting that cross-repository metadata heterogeneity blocks automated analysis despite the growing need for systematic reviews, reproducibility infrastructure, and ecosystem-wide dataset quality assessment. Practitioners face a critical barrier: no automated way to determine if documentation from HuggingFace, OpenML, and UCI is semantically comparable. Can lifecycle-stage functional roles—motivation, collection practices, responsible AI concerns—be discovered automatically from heterogeneous metadata text?

Our results establish that automated lifecycle detection is feasible, but requires supervised rather than unsupervised approaches. We tested the hypothesis that lifecycle stages would manifest as unsupervised clusters in semantic embedding space due to distributional regularities (lexical co-occurrence, normative modality, value-structural patterns). Instead, we found a striking asymmetry: linear probes achieve 97-100% accuracy (proving lifecycle signals are strongly encoded) while K-means clustering achieves NMI=0.02 (96% below threshold, essentially random). This **supervised-unsupervised gap** is the paper's central empirical contribution, revealing lifecycle separability as a supervised signal carrier requiring task-specific amplification—not an unsupervised emergent structure waiting to be discovered.

Three lines of evidence converge on this conclusion. **First**, inter-annotator agreement (κ=0.645) validates lifecycle categories as operationally stable constructs—annotators consistently identify functional roles despite schema heterogeneity. **Second**, linear probes generalize across repositories (79.6% overall accuracy, 82.7% HF → 76.0% UCI) with only modest degradation, proving distributional signatures transcend repository-specific conventions. **Third**, unsupervised clustering fails catastrophically (NMI=0.0229) despite these strong supervised signals, with baseline comparisons showing semantic embeddings provide minimal advantage (0.0129 gap over permutation, 91% below ≥0.15 threshold). The failure is not representational (embeddings lack information) but algorithmic (K-means cannot handle 11:1 class imbalance and repository heterogeneity).

Mechanistic attribution identifies **severe class imbalance** (8.3% Responsible AI fields vs. 91.7% General Information) as the primary bottleneck. K-means assumes balanced, well-separated clusters; extreme imbalance causes geometric compression of the minority class, leading to trivial majority-class assignments. Repository stratification corroborates this: UCI's 14% RAI prevalence (less severe 6:1 ratio) yields NMI=0.39—30x better than HuggingFace's 5.3% (18:1 ratio)—though still failing the 0.60 absolute threshold. Even the "best case" repository-specific clustering fails, establishing unsupervised discovery as algorithmically infeasible for highly imbalanced documentation metadata.

The supervised-unsupervised gap we identify is not a failure but a **boundary condition**: it clarifies what semantic embeddings can and cannot achieve for documentation metadata. Lifecycle structure exists (97-100% probe accuracy proves it), but manifests as a supervised signal requiring learned weighting to amplify minority classes—not a natural clustering detectable via geometric proximity alone. This distinction reshapes the feasibility landscape for automated metadata mapping. The question is no longer "can lifecycle be detected without labels?" (answer: no, unsupervised clustering fails) but "how many labels suffice for reliable cross-repository generalization?" (answer: preliminary evidence suggests 60 scaffolded HF samples enable 76% UCI accuracy—fewer labels may suffice with active learning or meta-learning).

Our negative result—unsupervised clustering failure despite strong supervised signals—has positive deployment implications. **It redirects research effort toward tractable problems**: few-shot label propagation, active learning to minimize annotation budgets, and semi-supervised methods that bootstrap from small labeled sets. Early experiments suggest 10-20 examples per repository may enable cross-repository lifecycle detection via probe-based classifiers—a modest annotation budget that makes systematic metadata assessment feasible at ecosystem scale.

Beyond proof-of-concept, we envision an **ecosystem-wide metadata quality dashboard** powered by semi-supervised lifecycle detectors. Repository maintainers could automatically audit documentation completeness, flagging datasets lacking responsible AI fields (ethical considerations, limitations, prohibited uses). Longitudinal tracking would reveal documentation practice evolution—is RAI adoption increasing post-2023 regulatory focus? Researchers conducting meta-analyses could efficiently filter datasets by lifecycle completeness across repositories, enabling systematic reproducibility studies currently infeasible due to manual mapping bottlenecks.

The path forward builds on our validated supervised signal. Immediate next steps include (1) testing class-imbalance-aware clustering (weighted K-means, DBSCAN) to determine if algorithmic improvements suffice or supervision is fundamental, (2) minimizing few-shot annotation budgets via active learning (selecting maximally informative examples), (3) explaining UCI's 30x better performance through controlled experiments isolating balance from vocabulary effects, and (4) extending beyond 2-tier granularity via hierarchical classification (first General vs. RAI, then subcategories).

Lifecycle-stage functional separability is real—inter-annotator agreement, linear separability, and cross-repository generalization prove it. Unsupervised discovery is infeasible—K-means clustering fails despite strong signals. But supervised detection works—97-100% probe accuracy establishes distributional signatures are exploitable with task-specific methods. This boundary clarifies deployment requirements: automated cross-repository metadata mapping needs labels, but not many. How few? That is the next question, and the most impactful one for enabling ecosystem-scale documentation quality assessment. The signal exists; we now know how to amplify it.
