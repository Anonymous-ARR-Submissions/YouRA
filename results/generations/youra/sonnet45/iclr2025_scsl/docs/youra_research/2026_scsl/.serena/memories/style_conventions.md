Conventions observed: Python modules use snake_case names and mostly small top-level scripts/functions with docstrings. Configuration is stored in uppercase dicts such as DATA_CONFIG and ENVIRONMENT_CONFIG. Research scripts prefer explicit paths, checkpoint/result directories, and readable print-based progress logs. Tests live alongside code subtrees under tests/ or as top-level test_*.py files. Markdown documentation is extensive and records experiment phases, validation status, and paper assets.
