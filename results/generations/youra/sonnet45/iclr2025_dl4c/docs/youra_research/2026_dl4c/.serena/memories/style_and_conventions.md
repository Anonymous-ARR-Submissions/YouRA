Code style is straightforward Python scripting with module-level config dictionaries, snake_case names, light docstrings, and minimal abstraction. Experiment scripts log progress with numbered stages, save outputs under sibling `results/` and `figures/` directories, and rely on local relative imports within each hypothesis folder. Analysis code in `h-m-integrated/code` follows the same convention and uses small classes for loaders, analyzers, validators, and visualizers.
